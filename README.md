# virtual-meetings-manager
The app is being developed to help event organizers easily plan, manage and host virtual events.

# Сайт для планирования и управления виртуальными мероприятиями

#### Сайт будет иметь такие функции, как:
1. создание событий 
2. управление мероприятиями
3. регистрация
4. обработка платежей 

Целевой аудиторией этого сайта будут организаторы мероприятий и лица, желающие проводить виртуальные мероприятия. 

#### Будут подробно описаны следующие требования: 
1. Технические требования
2. Пользовательский интерфейс
3. Пользовательский опыт
4. Дизайн базы данных
5. Безопасность
7. Интеграция со сторонними инструментами
8. Тестирование и развертывание

## Технические требования:

#### Требования к оборудованию
* Для сайта потребуется веб-сервер с минимум 2 ГБ оперативной памяти и 100 ГБ дискового пространства.
* Для обработки трафика на сайт потребуется балансировщик нагрузки.
* Сайт будет размещен в облачной инфраструктуре для лучшей масштабируемости и надежности.
#### Требования к программному обеспечению
* Бэкенд сайта будет разработан с использованием фреймворка Java Spring.
* Фронтенд сайта будет разработан с использованием HTML, СSS и JavaSСript.
* Для хранения данных сайта будет использоваться СУБД MySQL
#### Совместимость с браузером
* Сайт будет совместим с последними версиями Сhrome, Firefox и MiСrosoft Edge.
#### Сторонние интеграции
* Платежные системы PayPal, Qiwi, Robokassa, для обработки платежей за регистрацию на мероприятия.
* Gmail - Почтовая служба, для отправки зарегистрированным пользователям напоминаний о событиях и обновлениях.
#### Безопасность
* У сайта будет SSL-шифрование для обеспечения безопасности пользовательских данных во время передачи.
* У сайта будут присутствовать надлежащие меры аутентификации и авторизации для предотвращения несанкционированного доступа к сайту и его данным.
* У сайта будут проводиться регулярные обновления безопасности и исправления, чтобы предотвратить использование любых уязвимостей.

## Пользовательский интерфейс
#### 1. Навигация
У сайта будет интуитивно понятное навигационное меню, которое позволит пользователям легко получать доступ к различным разделам сайта. Навигационное меню будет расположено в верхней части страницы и будет доступно с любой страницы сайта.
#### 2. Домашняя страница
На главной странице будет баннер с призывом к действию, призывающий пользователей создать мероприятие. На главной странице также будет отображаться список избранных событий, панель поиска для поиска событий и список категорий для фильтрации событий.
#### 3. Создание события
Процесс создания события будет простым и понятным. Пользователи смогут вводить сведения о мероприятии, такие как название, описание, дата, время, место и стоимость билета. Пользователи также смогут загружать изображения и видео для продвижения своих мероприятий.
#### 4. Управление событиями
Пользователи смогут управлять своими событиями с единой панели управления. Панель управления событиями будет отображать список событий, их статус и количество участников. Пользователи смогут редактировать, удалять или продвигать свои события с панели управления.
#### 5. Присоединение к мероприятию
Пользователи смогут зарегистрироваться на мероприятия прямо со страницы мероприятия. Процесс регистрации будет безопасным и потребует от пользователей ввести свою личную информацию и платежные реквизиты.
#### 6. Страница событий
На странице мероприятия будут отображаться все подробности о мероприятии, включая описание, дату, время, место и информацию о билетах. На странице мероприятия также будет отображаться карта с местом проведения мероприятия и список зарегистрированных участников. Пользователи смогут оставлять комментарии на странице мероприятия и делиться событием в социальных сетях.
#### 7. Профиль пользователя
У пользователей будет личный профиль, где они смогут управлять своей информацией и просматривать историю своих событий. В профиле пользователя также будет отображаться оценка репутации пользователя, которая будет рассчитываться на основе количества мероприятий, которые они посетили, и их отзывов от организаторов мероприятий.
#### 8. Поиск и фильтрация
Сайт будет иметь поисковую систему, которая позволит пользователям находить события по ключевым словам, местоположению, дате и категории. На сайте также будет система фильтрации, которая позволит пользователям уточнять результаты поиска на основе таких критериев, как тип события, цена билета и популярность.
#### 9. Справка и поддержка
На сайте будет специальный раздел справки и поддержки, где пользователи смогут найти ответы на распространенные вопросы и получить помощь по техническим вопросам. Раздел справки и поддержки также будет включать контактную форму для отправки запросов в службу поддержки.
#### 10. Отзывы пользователей
На сайте будет реализована система сбора отзывов и предложений пользователей. Пользователи смогут оставлять отзывы через форму на сайте или по ссылке обратной связи в электронных письмах с уведомлениями.
#### 11. Обработка ошибок
Сайт будет иметь надежные механизмы обработки ошибок, чтобы обеспечить бесперебойную работу пользователей. В случае ошибок или сбоев системы на сайте будут отображаться понятные сообщения об ошибках и рекомендации по устранению проблемы.

#### 1. Удобство использования
Сайт будет иметь простой, интуитивно понятный и удобный интерфейс. Навигация будет четкой и простой, а общие действия и функции будут отображаться на видном месте.

#### 3. Персонализация
Пользователи смогут настроить сайт под свои нужды, включая язык, уведомления и предпочтения событий. Сайт запоминает предпочтения пользователя и предоставляет персонализированный опыт каждый раз, когда пользователь посещает сайт.
#### 4. Адаптивность
Сайт будет оптимизирован для просмотра на разных устройствах и размерах экрана. сайт будет протестирован на настольных компьютерах, планшетах и мобильных устройствах, чтобы обеспечить бесперебойную работу пользователей.

Сайт будет доступен для пользователей с ограниченными возможностями, в том числе с нарушениями зрения, слуха и моторики. Сайт будет соответствовать стандартам доступности, таким как WСAG 3, и будет проверен на доступность с использованием автоматизированных инструментов и ручного тестирования.

#### 10. Производительность
Сайт будет оптимизирован для повышения производительности с упором на быстрое время загрузки и отзывчивую навигацию. Сайт будет протестирован на производительность с использованием таких инструментов, как GTmetrix и Google PageSpeed Insights.
#### 11. Эстетика

#### 12. Общение
Пользователи смогут общаться с другими пользователями через сайт, в том числе посредством участия в мероприятиях, комментариев и обмена сообщениями.
#### Визуализация данных

#### 7. Настраиваемые шаблоны событий: 
На сайте будут представлены настраиваемые шаблоны событий, которые позволят пользователям быстро и легко создавать виртуальные события с последовательным и профессиональным внешним видом. Настраиваемые шаблоны событий будут настраиваемыми и могут быть адаптированы для удовлетворения потребностей отдельных событий. 
#### 8. Доступность: 
Сайт будет доступен и пригоден для использования пользователями с ограниченными возможностями, включая пользователей с нарушениями зрения, слуха и моторики. Сайт будет соответствовать  стандартам доступности WCAG 3. 
#### 9. Мобильная совместимость
Сайт будет разработан для совместимости с целым рядом мобильных устройств, включая смартфоны и планшеты. Удобный для мобильных устройств дизайн обеспечит единообразный пользовательский интерфейс для пользователей, заходящих на сайт с мобильных устройств. 
#### 10. Интеграция с внешними сервисами
Сайт будет обеспечивать интеграцию с внешними сервисами, такими как платформы электронного маркетинга и социальные сети, чтобы обеспечить бесперебойную и эффективную рекламу мероприятия и общение с пользователями.
#### 11. Панель управления событиями
На сайте будет представлена комплексная панель управления событиями, которая позволит пользователям управлять всеми аспектами виртуальных событий из единого интерфейса. Панель управления событиями будет предоставлять обновления, информацию и аналитику в режиме реального времени, чтобы помочь пользователям принимать обоснованные решения о планировании и управлении событиями. 
#### 12. Уведомления пользователей
Сайт будет предоставлять пользователям уведомления, чтобы информировать пользователей о важных событиях, обновлениях и напоминаниях, связанных с виртуальными событиями. Пользовательские уведомления можно настраивать и адаптировать к потребностям и предпочтениям отдельных пользователей. 
#### 13. Многоязычная поддержка
Сайт будет поддерживать несколько языков, чтобы обеспечить единообразный пользовательский интерфейс для пользователей по всему миру. Многоязычная поддержка будет включать в себя выбор языка, перевод пользовательского контента и локализацию форматов даты и времени. 
#### 14. Служба поддержки
Сайт будет обеспечивать всестороннюю поддержку, чтобы помочь пользователям с вопросами, проблемами и проблемами, связанными с виртуальными событиями и самим сайтом. Поддержка клиентов будет доступна по нескольким каналам, включая электронную почту, телефон и чат. 
#### 15. Управление пользователями
Сайт будет предоставлять систему управления пользователями, которая позволит пользователям создавать и управлять своими профилями и участниками мероприятия. Управление пользователями будет включать в себя параметры для импорта и экспорта данных об участниках, управления разрешениями пользователей и настройки пользовательских форм регистрации событий. 
#### 16. Аналитика событий
Сайт будет предоставлять аналитику событий в режиме реального времени, чтобы помочь пользователям оценивать успех своих виртуальных мероприятий и принимать решения на основе данных. Аналитика событий будет включать такие показатели, как посещаемость и вовлеченность. 
#### 17. Настройка события
На сайте будут представлены широкие возможности настройки, позволяющие пользователям создавать уникальные и запоминающиеся виртуальные мероприятия для своих клиентов. Параметры настройки будут включать возможность добавления элементов брендинга, настраиваемых страниц событий и URL-адресов настраиваемых событий. 
#### 18. Процесс оплаты
Сайт предоставит варианты обработки платежей, чтобы пользователи могли принимать платежи за виртуальные мероприятия и продажу билетов. Варианты обработки платежей будут включать поддержку нескольких платежных систем (PayPal, Qiwi, Robokassa), безопасную обработку платежей и обновление транзакций в режиме реального времени.
